# git am

> Применять файлы патчей и создавать коммиты. Полезно при получении коммитов по электронной почте.
> Смотрите также: `git format-patch`.
> Больше информации: <https://git-scm.com/docs/git-am>.

- Применить и зафиксировать изменения из локального файла патча:

`git am {{путь/к/файлу.patch}}`

- Применить и зафиксировать изменения из удалённого файла патча:

`curl {{[-L|--location]}} {{https://example.com/file.patch}} | git am`

- Прервать процесс применения файла патча:

`git am --abort`

- Применить как можно больше частей файла патча, сохраняя неудачные фрагменты (hunks) в файлы отклонений (.rej):

`git am --reject {{путь/к/файлу.patch}}`
