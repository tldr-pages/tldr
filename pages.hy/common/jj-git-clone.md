# jj git clone

> Ստեղծեք նոր ռեպո, որն ապահովված է Git ռեպո-ի կլոնով:.
> Նշում. Եթե `--colocate`-ը չի օգտագործվում, այն վավեր Git պահոց չէ, և `git` հրամանները չեն կարող օգտագործվել դրա վրա:.
> Լրացուցիչ տեղեկություններ. <https://docs.jj-vcs.dev/latest/cli-reference/#jj-git-clone>:.

- Ստեղծեք նոր ռեպո, որն ապահովված է Git ռեպո-ի կլոնով նոր գրացուցակում (կանխադրված գրացուցակը պահեստի անունն է).:

`jj git clone {{source}} {{path/to/directory}}`

- Ստեղծեք կլոն և օգտագործեք տրված անունը նոր ստեղծված հեռակառավարման համար.:

`jj git clone --remote {{remote_name}} {{source}}`

- Կլոնավորեք Git ռեպո՝ ստանալով միայն ամենավերջին 10 պարտավորությունները.:

`jj git clone --depth {{10}} {{source}}`

- Կլոնավորել Jujutsu ռեպո-ն Git ռեպո-ի հետ (թույլ տալով օգտագործել երկու `jj` և `git` հրամանները նույն գրացուցակում).:

`jj git clone --colocate {{source}}`
