# git վերականգնում

> Հետարկել փոփոխությունները կամ հանել փոփոխությունները՝ ընթացիկ Git HEAD-ը վերակայելով նշված վիճակին:.
> Եթե ճանապարհն անցնում է, այն աշխատում է որպես «անբեմականացում»; եթե commit hash-ը կամ մասնաճյուղը փոխանցվում է, այն աշխատում է որպես «uncommit»:.
> Լրացուցիչ տեղեկություններ. <https://git-scm.com/docs/git-reset>:.

- Բեմադրել ամեն ինչ.:

`git reset`

- Բեմադրել հատուկ ֆայլ(ներ).:

`git reset {{path/to/file1 path/to/file2 ...}}`

- Ինտերակտիվ կերպով հանել ֆայլի մասերը.:

`git reset {{[-p|--patch]}} {{path/to/file}}`

- Չեղարկել վերջին պարտավորությունը՝ պահպանելով դրա փոփոխությունները (և հետագա չհանձնված փոփոխությունները) ֆայլային համակարգում.:

`git reset HEAD~`

- Չեղարկել վերջին երկու պարտավորությունները՝ ավելացնելով դրանց փոփոխությունները ինդեքսի վրա, այսինքն՝ բեմադրված կատարման համար.:

`git reset --soft HEAD~2`

- Հրաժարվեք բոլոր չհաստատված փոփոխություններից՝ բեմականացված կամ ոչ փուլային (միայն չբեմադրված փոփոխությունների համար օգտագործեք `git checkout`):

`git reset --hard`

- Վերականգնել պահեստը տվյալ commit-ի վրա՝ հրաժարվելով կատարված, փուլային և չստուգված փոփոխություններից այդ ժամանակից ի վեր.:

`git reset --hard {{commit}}`
