# rmlint

> Գտեք տարածության թափոններ և այլ կոտրված բաներ ձեր ֆայլային համակարգում:.
> Լրացուցիչ տեղեկություններ. <https://rmlint.readthedocs.io/en/latest/rmlint.1.html>:.

- Ստուգեք դիրեկտորիաները կրկնօրինակված, դատարկ և կոտրված ֆայլերի համար.:

`rmlint {{path/to/directory1 path/to/directory2 ...}}`

- Ստուգեք որոշակի չափից ավելի մեծ կրկնօրինակներ, ցանկալի է ֆայլերը պահել պիտակավորված գրացուցակներում (կրկնակի կտրվածքից հետո).:

`rmlint {{[-s|--size]}} {{1MB}} {{path/to/directory}} // {{path/to/original_directory}}`

- Ստուգեք տիեզերական վատնողների համար՝ ամեն ինչ պահելով չպիտակավորված գրացուցակներում.:

`rmlint {{[-k|--keep-all-untagged]}} {{path/to/directory}} // {{path/to/original_directory}}`

- Ջնջել կրկնօրինակ ֆայլերը, որոնք գտնվել են `rmlint`-ի կատարմամբ.:

`./rmlint.sh`

- Գտեք կրկնօրինակ գրացուցակի ծառեր՝ հիմնված տվյալների վրա՝ անտեսելով անունները.:

`rmlint {{[-D|--merge-directories]}} {{path/to/directory}}`

- Ցածր [d] խորության ֆայլերը նշեք որպես բնօրինակներ, փողկապի վրա ընտրեք ավելի կարճ [l]երկարություն.:

`rmlint {{[-S|--rank-by]}} {{dl}} {{path/to/directory}}`

- Գտեք ֆայլեր նույնական ֆայլի անունով և բովանդակությամբ և կապեք, այլ ոչ թե ջնջեք կրկնօրինակները.:

`rmlint {{[-c|--config]}} sh:link {{[-b|--match-basename]}} {{path/to/directory}}`

- Օգտագործեք `data` որպես հիմնական գրացուցակ: Գտեք միայն կրկնօրինակները կրկնօրինակում, որոնք նույնպես `data`-ում են: Մի ջնջեք `data` ֆայլը՝:

`rmlint {{path/to/backup}} // {{path/to/data}} {{[-k|--keep-all-tagged]}} {{[-m|--must-match-tagged]}}`
