# git bisect

> Օգտագործեք երկուական որոնում՝ գտնելու այն commit-ը, որը սխալ է ներկայացրել:.
> Git-ը ինքնաբերաբար հետ ու առաջ է ցատկում կատարման գծապատկերում, որպեսզի աստիճանաբար նեղացնի անսարք կատարումը:.
> Լրացուցիչ տեղեկություններ. <https://git-scm.com/docs/git-bisect>:.

- Սկսեք բիսեկ սեսիա կատարման տիրույթում, որը սահմանափակված է հայտնի խելագարված առաջադրանքով և հայտնի մաքուր (սովորաբար ավելի հին) մեկով.:

`git bisect start {{bad_commit}} {{good_commit}}`

- Յուրաքանչյուր պարտավորության համար, որը ընտրում է `git bisect`, նշեք այն որպես «վատ» կամ «լավ»՝ խնդրի համար փորձարկելուց հետո.:

`git bisect {{good|bad}}`

- Ավարտեք բիսեկտի նիստը և վերադարձեք նախորդ ճյուղ.:

`git bisect reset`

- Բաց թողնել հանձնառությունը բիսեկտի ժամանակ (օրինակ, մեկը, որը ձախողում է թեստերը մեկ այլ խնդրի պատճառով).:

`git bisect skip`

- Սկսեք երկշերտ նստաշրջան՝ հաշվի առնելով միայն կոնկրետ ֆայլը կամ գրացուցակը փոփոխող պարտավորությունները.:

`git bisect start {{bad_commit}} {{good_commit}} -- {{path/to/file_or_directory}}`

- Ավտոմատացրեք երկհատվածի գործընթացը՝ օգտագործելով թեստային սկրիպտ, որը `exit` ունի 0՝ «լավ» և ոչ զրոյական՝ «վատ»:

`git bisect run {{path/to/test_script}} {{optional_script_arguments}}`

- Ցուցադրել մատյան, թե ինչ է արվել մինչ այժմ.:

`git bisect log`

- Ցույց տալ մնացած թեկնածուի պարտավորությունները, որոնք պետք է ստուգվեն.:

`git bisect visualize`
