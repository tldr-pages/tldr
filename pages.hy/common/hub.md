# հանգույց

> Git-ի փաթաթան, որն ավելացնում է հրամաններ GitHub-ի վրա հիմնված նախագծերի հետ աշխատելու համար:.
> Եթե կարգավորվում է `hub alias`-ի հրահանգով, կարելի է օգտագործել `git` `hub` հրամանները գործարկելու համար:.
> Լրացուցիչ տեղեկություններ. <https://hub.github.com/hub.1.html>:.

- Կլոնավորեք պահեստը, օգտագործելով իր slug-ը (սեփականատերերը կարող են բաց թողնել օգտվողի անունը).:

`hub clone {{username}}/{{repo_name}}`

- Ստեղծեք ընթացիկ պահոցի պատառաքաղ (կլոնավորված այլ օգտվողից) ձեր GitHub պրոֆիլի ներքո.:

`hub fork`

- Ներկայիս տեղական մասնաճյուղը մղեք GitHub և դրա համար PR ստեղծեք սկզբնական պահոցում.:

`hub push {{remote_name}} && hub pull-request`

- Ստեղծեք ընթացիկ (արդեն մղված) ճյուղի PR՝ կրկին օգտագործելով առաջին կոմիտից ստացված հաղորդագրությունը.:

`hub pull-request --no-edit`

- Ստեղծեք նոր մասնաճյուղ՝ ձգման հարցման բովանդակությամբ և անցեք դրան.:

`hub pr checkout {{pr_number}}`

- Ներբեռնեք ընթացիկ (միայն տեղական) պահոցը ձեր GitHub հաշվին.:

`hub create`

- Վերցրեք Git օբյեկտները վերևից և թարմացրեք տեղական մասնաճյուղերը.:

`hub sync`
