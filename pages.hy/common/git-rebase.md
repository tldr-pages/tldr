# git rebase

> Վերակիրառել պարտավորությունները մի ճյուղից մյուս մասնաճյուղի վերևում:.
> Սովորաբար օգտագործվում է ամբողջ ճյուղը մեկ այլ բազա «տեղափոխելու» համար՝ ստեղծելով նոր վայրում կատարվող պարտավորությունների պատճենները:.
> Լրացուցիչ տեղեկություններ. <https://git-scm.com/docs/git-rebase>:.

- Վերահաստատեք ընթացիկ մասնաճյուղը մեկ այլ նշված ճյուղի վրա.:

`git rebase {{new_base_branch}}`

- Սկսեք ինտերակտիվ վերահաշվարկ, որը թույլ է տալիս վերադասավորել, բաց թողնել, համադրել կամ փոփոխել պարտավորությունները.:

`git rebase {{[-i|--interactive]}} {{target_base_branch_or_commit_hash}}`

- Շարունակեք վերահիմնավորումը, որն ընդհատվել էր միաձուլման ձախողման պատճառով՝ հակասական ֆայլերը խմբագրելուց հետո.:

`git rebase --continue`

- Շարունակեք վերանայումը, որը դադարեցվել էր միաձուլման կոնֆլիկտների պատճառով՝ բաց թողնելով հակամարտող պարտավորությունը.:

`git rebase --skip`

- Դադարեցրեք վերանայման գործընթացը (օրինակ, եթե այն ընդհատվում է միաձուլման հակամարտությամբ).:

`git rebase --abort`

- Ընթացիկ ճյուղի մի մասը տեղափոխեք նոր բազա՝ ապահովելով հին բազան սկսելու համար.:

`git rebase --onto {{new_base}} {{old_base}}`

- Կրկին կիրառեք վերջին 5 պարտավորությունները տեղում՝ դադարեցնելով, որպեսզի դրանք վերադասավորվեն, բաց թողնվեն, համակցվեն կամ փոփոխվեն.:

`git rebase {{[-i|--interactive]}} {{HEAD~5}}`

- Ավտոմատ լուծեք ցանկացած կոնֆլիկտ՝ նախընտրելով աշխատանքային մասնաճյուղի տարբերակը (`theirs` հիմնաբառը այս դեպքում հակադարձ նշանակություն ունի).:

`git rebase {{[-X|--strategy-option]}} theirs {{branch_name}}`
