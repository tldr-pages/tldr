# ssh

> Secure Shell-ը արձանագրություն է, որն օգտագործվում է հեռավոր համակարգեր ապահով մուտք գործելու համար:.
> Այն կարող է օգտագործվել հեռավոր սերվերի վրա գրանցման կամ հրամանների կատարման համար:.
> Լրացուցիչ տեղեկություններ. <https://man.openbsd.org/ssh>:.

- Միացեք հեռավոր սերվերին.:

`ssh {{username}}@{{remote_host}}`

- Միացեք որոշակի [i] ինքնությամբ (մասնավոր բանալի) հեռավոր սերվերին.:

`ssh -i {{path/to/key_file}} {{username}}@{{remote_host}}`

- Միացեք հեռավոր սերվերին `10.0.0.1` IP-ով և օգտագործելով որոշակի [p]ort (Նշում. `10.0.0.1`-ը կարող է կրճատվել մինչև `10.1`):

`ssh {{username}}@10.0.0.1 -p {{2222}}`

- Գործարկեք հրաման հեռավոր սերվերի վրա [t]ty հատկացումով, որը թույլ է տալիս փոխազդել հեռակառավարման հրամանի հետ.:

`ssh {{username}}@{{remote_host}} -t {{command}} {{command_arguments}}`

- SSH թունելավորում. [D]ինամիկ պորտի վերահասցեավորում (SOCKS վստահված անձ `localhost:1080`-ում):

`ssh -D {{1080}} {{username}}@{{remote_host}}`

- SSH թունելավորում. փոխանցեք որոշակի նավահանգիստ (`localhost:9999` դեպի `example.org:80`), ինչպես նաև անջատելով հեռակա հրամանների կեղծ[T]ty հատկացումը և կատարումը[N]:

`ssh -L {{9999}}:{{example.org}}:{{80}} -N -T {{username}}@{{remote_host}}`

- SSH [J]umping. Jumphost-ի միջոցով միացեք հեռավոր սերվերին (բազմաթիվ թռիչքներ կարող են նշվել՝ բաժանված ստորակետերով).:

`ssh -J {{username}}@{{jump_host}} {{username}}@{{remote_host}}`

- Փակել կախովի նիստը.:

`<Enter><~><.>`
