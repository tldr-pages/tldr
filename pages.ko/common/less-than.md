# <

> 데이터를 표준 입력(`stdin`)으로 리다이렉트.
> 더 많은 정보: <https://gnu.org/software/bash/manual/bash.html#Redirecting-Input>.

- 파일 내용을 `stdin`으로 전달 (`cat file.txt |`와 동일한 효과):

`{{명령어}} < {{경로/대상/파일.txt}}`

- here document를 생성하여 `stdin`으로 전달 (여러 줄 입력):

`{{명령어}} << {{EOF}} <Enter> {{여러줄_텍스트}} <Enter> {{EOF}}`

- here string을 생성하여 `stdin`으로 전달 (`echo string |`와 동일한 효과):

`{{명령어}} <<< {{문자열}}`

- 파일을 입력으로 처리하고 결과를 다른 파일에 저장:

`{{명령어}} < {{경로/대상/파일.txt}} > {{경로/대상/파일2.txt}}`

- here document를 파일에 기록:

`cat << {{EOF}} > {{경로/대상/파일.txt}} <Enter> {{여러줄_텍스트}} <Enter> {{EOF}}`

- 앞의 탭 무시 (들여쓰기가 있는 스크립트에서는 유용하지만, 공백에는 동작하지 않음):

`cat <<- {{EOF}} > {{경로/대상/파일.txt}} <Enter> {{여러줄_데이터}} <Enter> {{EOF}}`

- 명령 출력 결과를 파일 디스크립터 형태로 다른 프로그램에 전달 (참고: 앞의 예제들과는 달리, 이 방식은 입력을 표준 입력으로 전달하지 않고 인자를 `/dev/fd/63`와 같은 파일 경로로 치환됨):

`diff <({{명령어1}}) <({{명령어2}})`

- 지속적인 파일 디스크립터 열기:

`exec {{3}}<{{경로/대상/파일}}`
