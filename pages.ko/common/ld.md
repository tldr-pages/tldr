# ld

> 오브젝트 파일들을 연결해 실행 파일 생성.
> 더 많은 정보: <https://sourceware.org/binutils/docs/ld.html#Options>.

- 의존성 없이 특정 오브젝트 파일을 실행 파일로 링크:

`ld {{경로/대상/파일.o}} {{[-o|--output]}} {{경로/대상/출력_실행파일}}`

- 여러 오브젝트 파일을 함께 링크하여 실행 파일 생성:

`ld {{경로/대상/파일1.o 경로/대상/파일2.o ...}} {{[-o|--output]}} {{경로/대상/출력_실행파일}}`

- 특정 대상 에뮬레이션을 사용하여 오브젝트 파일 링크:

`ld {{경로/대상/파일.o}} -m {{대상_에뮬레이션}} {{[-o|--output]}} {{경로/대상/출력_실행파일}}`

- x86_64 프로그램을 glibc와 동적 링크 (파일 경로는 시스템에 따라 달라질 수 있음):

`ld {{[-o|--output]}} {{경로/대상/출력_실행파일}} {{[-I|--dynamic-linker]}} {{/lib/ld-linux-x86-64.so.2}} {{/lib/crt1.o}} {{/lib/crti.o}} -lc {{경로/대상/파일.o}} {{/lib/crtn.o}}`
