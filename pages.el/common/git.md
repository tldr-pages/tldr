# git

> Κατανεμημένο σύστημα ελέγχου εκδόσεων.
> Μερικές υποεντολές όπως `commit`, `add`, `branch`, `switch`, `push`, κ.λπ. έχουν τη δική τους τεκμηρίωση χρήσης.
> Περισσότερες πληροφορίες: <https://git-scm.com/docs/git>.

- Δημιουργία ενός άδειου αποθετηρίου Git:

`git init`

- Κλωνοποίηση ενός remote αποθετηρίου Git από το διαδίκτυο:

`git clone {{https://example.com/repo.git}}`

- Προβολή της κατάστασης του τοπικού αποθετηρίου:

`git status`

- Προετοιμασία (stage) όλων ([A]ll) των αλλαγών για υποβολή (commit):

`git add {{[-A|--all]}}`

- Υποβολή (commit) αλλαγών στο ιστορικό εκδόσεων με μήνυμα ([m]essage):

`git commit {{[-m|--message]}} {{message_text}}`

- Αποστολή (push) τοπικών υποβολών σε remote αποθετήριο:

`git push`

- Λήψη (pull) τυχόν αλλαγών που έγιναν σε remote αποθετήριο:

`git pull`

- Επαναφορά όλων όπως ήταν στην τελευταία υποβολή (commit) και αναγκαστικός ([f]orce) καθαρισμός αρχείων:

`git reset --hard; git clean {{[-f|--force]}}`
